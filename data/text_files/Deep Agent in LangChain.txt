Deep Agent in LangChain: A Comprehensive Guide
Introduction
Deep Agent represents a significant advancement in the LangChain ecosystem, offering a sophisticated framework for building autonomous AI agents capable of complex reasoning, tool interaction, and multi-agent collaboration. Unlike traditional chatbots or simple LLM wrappers, Deep Agents are designed to handle intricate workflows, manage persistent state, and execute advanced tasks with human-like decision-making capabilities.

This comprehensive guide explores the architecture, features, implementation details, and best practices for leveraging Deep Agents in your applications.

What is a Deep Agent?
A Deep Agent is an intelligent autonomous system built on LangChain that combines language models with a sophisticated toolkit for reasoning, planning, and execution. At its core, a Deep Agent:

Requires LLM Tool Calling Support: Deep Agents mandate that the underlying language model supports function calling capabilities, enabling the agent to invoke tools programmatically
Manages Complex Workflows: Handles multi-step processes, file operations, and computational tasks
Maintains State and Memory: Persists information across interactions using checkpointing and memory systems
Coordinates Multiple Agents: Can delegate tasks to subagents for specialized problem-solving
Operates in Sandboxed Environments: Executes code and shell commands safely within isolated execution contexts
Architecture Overview
Core Components
1. State Management
Deep Agents operate through a sophisticated state machine that tracks:

Messages and conversation history
Tool execution results
File system state
Memory and context
Interruption points for human-in-the-loop workflows
The agent's state is managed through a CompiledStateGraph, which provides deterministic execution and reproducibility.

2. Built-in Tools
By default, Deep Agents come equipped with a comprehensive toolkit:

Todo Management: write_todos for managing task lists
File Operations: ls, read_file, write_file, edit_file, glob, grep for filesystem manipulation
Shell Execution: execute for running shell commands in a sandbox environment
Subagent Invocation: task for delegating work to specialized subagents
3. Backend Systems
Deep Agents support multiple backend implementations:

StateBackend: In-memory state management for stateless deployments
StoreBackend: Uses LangGraph's store abstraction for persistent storage
FilesystemBackend: Direct filesystem access with security controls
DaytonaSandbox: Isolated execution environment for secure code execution
CompositeBackend: Routes different paths to different backends
4. Middleware Layer
Middleware components enable advanced features:

SummarizationMiddleware: Manages context length by intelligently summarizing conversation history
SummarizationToolMiddleware: Coordinates summarization with tool interactions
SkillSandboxSyncMiddleware: Synchronizes skills between storage backends and sandbox environments
Creating a Deep Agent: Basic Setup
Simple Agent with Tool Support
The most straightforward approach to creating a Deep Agent:


# pip install -qU deepagents langchain-openai
from deepagents import create_deep_agent

def get_weather(city: str) -> str:
    """Get weather for a given city."""
    return "It's always sunny in {city}!"

agent = create_deep_agent(
    model="openai:gpt-4",
    tools=[get_weather],
    system_prompt="You are a helpful assistant",
)

# Run the agent
result = agent.invoke(
    {"messages": [{"role": "user", "content": "what is the weather in sf"}]}
)
This example demonstrates:

Model specification with provider prefix
Custom tool definition
System prompt configuration
Agent invocation with message input

Advanced Configuration
The create_deep_agent() Function
The create_deep_agent() function is highly configurable with numerous parameters:


from deepagents import create_deep_agent
from langgraph.checkpoint.memory import MemorySaver

agent = create_deep_agent(
    # Core Configuration
    model="openai:gpt-4",                          # LLM to use
    tools=[my_tool_1, my_tool_2],                 # Custom tools
    system_prompt="Custom system instructions",    # Agent behavior
    
    # Agent Capabilities
    subagents=[specialized_agent_1],              # Subagents for delegation
    skills=["/skills/path"],                      # Skill definitions
    memory=["/memory/preferences.md"],            # Memory files
    
    # Execution Environment
    backend=filesystem_backend,                   # Execution backend
    permissions=[filesystem_permission],          # File system access controls
    
    # Middleware and Interception
    middleware=[summarization_mw],                # Processing middleware
    interrupt_on={                                # Human-in-the-loop
        "write_file": True,
        "edit_file": True,
        "read_file": False
    },
    
    # Response and Context
    response_format=structured_schema,            # Response format validation
    context_schema=custom_context_type,           # Context type definition
    
    # Persistence and Caching
    checkpointer=MemorySaver(),                  # State persistence
    store=base_store,                            # Data storage
    cache=base_cache,                            # Response caching
    
    # Debugging
    debug=True,                                  # Debug mode
    name="my_agent"                              # Agent identifier
)

Working with Skills
Skills are specialized knowledge domains that Deep Agents can access to handle specific types of queries. Skills are typically defined in markdown files with structured metadata.

Basic Skill Implementation

from urllib.request import urlopen
from deepagents import create_deep_agent
from deepagents.backends import StateBackend, StoreBackend, FilesystemBackend
from deepagents.backends.utils import create_file_data
from langgraph.checkpoint.memory import MemorySaver

# Example: Using a skill from a remote URL
skill_url = "https://raw.githubusercontent.com/langchain-ai/deepagents/refs/heads/main/libs/cli/examples/skills/langgraph-docs/SKILL.md"
with urlopen(skill_url) as response:
    skill_content = response.read().decode('utf-8')

checkpointer = MemorySaver()

skills_files = {
    "/skills/langgraph-docs/SKILL.md": create_file_data(skill_content)
}

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    skills=["/skills/"],
    checkpointer=checkpointer,
)

result = agent.invoke(
    {
        "messages": [
            {
                "role": "user",
                "content": "What is langgraph?"
            }
        ],
        "files": skills_files
    },
    config={"configurable": {"thread_id": "12345"}},
)

Context-Aware Memory and Personalization
Deep Agents can maintain isolated, user-specific memories and preferences using the store backend with namespace isolation:


from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from deepagents.backends.utils import create_file_data
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()

# Seed preferences for Alice
store.put(
    ("user-alice",),
    "/memories/preferences.md",
    create_file_data("""## Preferences
- Likes concise bullet points
- Prefers Python examples
"""),
)

# Seed preferences for Bob
store.put(
    ("user-bob",),
    "/memories/preferences.md",
    create_file_data("""## Preferences
- Likes detailed explanations
- Prefers TypeScript examples
"""),
)

# Seed a skill specific to Alice
store.put(
    ("user-alice",),
    "/skills/langgraph-docs/SKILL.md",
    create_file_data("""---
name: langgraph-docs
description: Fetch relevant LangGraph documentation.
---
Use the fetch_url tool to read docs...
"""),
)

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    memory=["/memories/preferences.md"],
    skills=["/skills/"],
    backend=lambda rt: CompositeBackend(
        default=StateBackend(rt),
        routes={
            "/memories/": StoreBackend(
                rt,
                namespace=lambda rt: (rt.server_info.user.identity,),
            ),
            "/skills/": StoreBackend(
                rt,
                namespace=lambda rt: (rt.server_info.user.identity,),
            ),
        },
    ),
    store=store,
)
In a deployed environment, each authenticated request automatically isolates memories and skills by user identity.

Context Management with Summarization Middleware
For long-running agents dealing with extended conversations, Deep Agents include summarization middleware to manage token consumption:


from deepagents import create_deep_agent
from deepagents.middleware.summarization import (
    SummarizationMiddleware,
    SummarizationToolMiddleware,
)
from deepagents.backends import FilesystemBackend

backend = FilesystemBackend(root_dir="/data")

# Configure summarization to trigger at 85% of context limit
# and keep 10% of context after summarization
summ = SummarizationMiddleware(
    model="gpt-4o-mini",
    backend=backend,
    trigger=("fraction", 0.85),  # Trigger at 85% capacity
    keep=("fraction", 0.10),     # Retain 10% after summary
)

tool_mw = SummarizationToolMiddleware(summ)

agent = create_deep_agent(
    middleware=[summ, tool_mw]
)
The summarization middleware:

Monitors context usage during execution
Triggers summarization when specified thresholds are reached
Preserves critical information while reducing token consumption
Works seamlessly with tool execution
Human-in-the-Loop Workflows
Deep Agents support interruption points for human review and approval:


from deepagents import create_deep_agent
from deepagents.backends.filesystem import FilesystemBackend
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=FilesystemBackend(root_dir="/Users/user/project"),
    interrupt_on={
        "write_file": True,    # Require approval: approve, edit, reject
        "read_file": False,    # No interrupts
        "edit_file": True,     # Require approval
        "execute": {           # Advanced config
            "resume_type": "approve",  # Auto-approve approach
        }
    },
    checkpointer=checkpointer,  # REQUIRED for human-in-the-loop
)
The checkpointer is essential for human-in-the-loop workflows, enabling the agent to pause, receive human feedback, and resume execution.

Subagents and Task Delegation
Deep Agents can coordinate with specialized subagents:


from deepagents import create_deep_agent, SubAgent

# Create specialized subagents
research_agent = SubAgent(
    name="research",
    description="Researches topics using web tools"
)

analysis_agent = SubAgent(
    name="analysis",
    description="Analyzes data and generates insights"
)

# Main agent coordinates their work
main_agent = create_deep_agent(
    model="openai:gpt-4",
    subagents=[research_agent, analysis_agent],
    system_prompt="You are a research coordinator..."
)
The agent can then use the task tool to delegate work to subagents.

from deepagents.backends import FilesystemPermission
